  • These are minimal changes to Wicket to allow other projects to work. In particular, it has refactored some concrete API classes to interfaces. There is also an experimental extension that encode the page/component info in URLs as an opaque ID. See org/apache/wicket/request/mapper/info/DefaultPageComponentInfoCodec.java for details.

wicklet-dsl

  • Domain Specific Languages (DSL) for HTML, CSS and scripts that provide an alternate way to generate the template DOM and stylesheets. Instead of writing literal text or a separate HTML file and parse it to a DOM: 
    <html>
    <head>...</head>
    <body>... <div wicket:id="test">...</div>...</body>
</html>
    you write this Java code: 
    html(
    head(...), 
    body(... component(WID.test).div(...) ...)
);
    that generates the DOM or component tree. Using DSL, you can leverage the Java programming language constructs as well as assists from a modern Java IDE. The DSL for HTML is typically used for creating the template DOM, either statically or dynamically. It can also be used for ahead of time generation of static web pages. The DSL for CSS is typically used for ahead of time generation of the CSS stylesheets. The simplistic DSL for scripts assists writing scripts in most readable form.

Wicklet/GWT integration

  • GWT is a Java to Javascript compiler. It also provide a client side UI framework. Unfortunately, it has its own event system. However, it support standard AJAX. With a little bit of interface code, it can works fine with Wicklet at the server side. See sf/wicklet/gwt/server/ajax/impl/GwtAjaxWickletTarget.java, sf/wicklet/gwt/server/ajax/impl/GwtAjaxXmlTarget.java, sf/wicklet/gwt/client/ajax/WickletAjax.java and related classes for details. With Wicklet/GWT integration, both the server side and the client side can be coded in Java.
  • Out of the box, GWT do not works well with HTML templates. But with a little bit of hack, it can. That opens up the possibility of progressive enhancement of a site that can works with/without GWT. See sf.wicklet.gwt.client.dsl.gwt.widgets.WickletGwt.WickletTemplateFlowPanel for more detail.
  • It is also possible to intermix Wicket components and GWT UI too. See sf/wicklet/site/gwt/login/client/WickletGwtSiteLogin.java where it use the WxSignInPanel component for login. See also sf/wicklet/site/gwt/server/pages/s/Wikicomment.java where GWT request and submit a form using AJAX while Wicket form rendering and processing is used at the server side.
