Sources

The sources are hosted at https://github.com/wicklet/ .

Building

  • To build from source, clone Wicklet.git, WickletSupport.git and WickletRelease.git.
  • Copy the customized artifacts, that are not available from public repositories, at wicklet-release/src/main/resources/repository/* to ~/.m2/repository/. The wicklet-release/src/main/resources/patches/ directory has the patch files for building the customized artifacts. Artifacts available from standard external repositories are not included. Either create a local cache or access to external repositories, like maven central, is required for the dependencies.
  • To build with eclipse (tested with eclipse-3.7.2), import all projects as existing maven projects. Update project configuration and perform maven clean and maven install in the wicklet-release project.
  • To build with maven (tested with maven-3.0.4), run mvn clean; mvn install in the wicklet-release project.
  • To run the arquillian and selenium tests, you need a tomcat7 installation in WickletGwtSiteTest/../opt/tomcat7/ with the manager webapp and a proper tomcat-users.xml. You can find a sample tomcat-users.xml in wicklet-release/src/main/resources/tomcat7/conf/. For selenium tests, you need a firefox profile with javascript and cookies enabled at WickletGwtSiteTest/../opt/firefox/7x16slsr.default/. Simply copy a firefox profile from you home directory would probably work.

Running

  • For a quick test, deploy the pre-built binary wicklet-release/src/main/resources/tomcat7/webapps/wicklet-site.war to a tomcat7 installation, with http port at 8080 and https port at 8443, and access http://localhost:8080/wicklet-site/ with firefox. The webapp contains the project documentation. Project documentation can also be found in the WickletGwtSite/src/main/webapp/wiki/ directory.
  • If built from source, deploy WickletGwtSite/target/WickletGwtSite-<version>.war as wicklet-site.war.
  • To try out the wiki, login as editor with password editor. To access the admin page, login as admin with password tomcat6. Note that, the web server need write access to the wicklet-site webapp directory in order for comment, wiki edit and database update to work.
